### Feature Engineering Execution Time

<CodeTabs tabs={["Python (Pandas)", "SQL (DuckDB)"]}>
  
  <div slot="tab-0">
```python
import pandas as pd

# Pandas uses a single thread and consumes heavy memory
df = pd.read_csv("previous_application.csv")
df['avg_credit'] = df.groupby('SK_ID_CURR')['AMT_CREDIT'].transform('mean')
```
  </div>

  <div slot="tab-1">
```sql
-- DuckDB executes the same logic in seconds using multi-threading
SELECT 
  SK_ID_CURR,
  AVG(AMT_CREDIT) OVER (PARTITION BY SK_ID_CURR) AS avg_credit
FROM 'previous_application.csv';
```
  </div>
</CodeTabs>

### XGBoost vs. Random Forest

<ComparisonGrid>
  <ComparisonBox title="XGBoost Advantages" type="pros">
    - Handles missing data automatically without manual imputation.
    - Built-in cross-validation at each iteration.
    - Extremely fast execution speed compared to traditional GBMs.
    - Tree pruning helps in preventing overfitting.
  </ComparisonBox>

  <ComparisonBox title="XGBoost Disadvantages" type="cons">
    - Requires more hyperparameter tuning than Random Forest.
    - Can be prone to overfitting if parameters (like depth) are not tuned well.
    - Harder to interpret compared to a single decision tree.
  </ComparisonBox>
</ComparisonGrid>

### Why DuckDB over Pandas for 300k+ Records?

<ComparisonRow>
<CompBox title="Pandas Approach" type="cons">
When engineering 140+ features using Pandas `apply()` and `groupby()` functions, the process consumed excessive RAM and ran on a single thread. It became a significant bottleneck in the pipeline.
</CompBox>

<CompBox title="DuckDB Approach" type="pros">
DuckDB seamlessly executed complex Window Functions directly on the CSV/Parquet files. It automatically utilized all CPU cores, dropping feature engineering time from minutes to mere seconds.
</CompBox>
</ComparisonRow>

### Mathematical Formulation
Tezindeki o kritik Model Predictive Control (MPC) maliyet fonksiyonunu (Cost Function) vurgulamak için `<MathFocus>` kullan. Arka planda o şık sigma (∑) işareti belirecek:

<MathFocus>
  The objective of the Model Predictive Controller is to minimize the cost function $J$ over the prediction horizon $N$:

  $$ J = \sum_{k=0}^{N-1} \left( \| x_{k+1} - r_{k+1} \|_{Q}^2 + \| u_k \|_{R}^2 \right) $$

</MathFocus>
